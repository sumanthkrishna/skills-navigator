# 06 — Reporting and Retro Insights

## Report Types

1. Story-level report
2. Pull-request-level report
3. Deployment-level report
4. Sprint-level retro report
5. Leadership summary
6. Team improvement backlog

## Story-Level Report Sections

```text
Summary
Timeline
Metric Breakdown
Bottleneck Classification
Recommended Actions
Raw Event Appendix
```

## Sprint-Level Retro Questions

- Which stories had the highest wait time?
- Which pipelines failed most often?
- Which repositories had the longest build time?
- Which PRs waited longest for review?
- Which deployments spent longest in queue?
- Which quality gates caused repeated rework?

## Recommended Retro Framing

Use language such as:

> The delivery system introduced 31 hours of wait time across this story.

Avoid language such as:

> The engineer only coded for 4 hours.
